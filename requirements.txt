# Sci-Oly Question Banks — Python dependencies
#
# Pinned to known-good minimums to prevent silent breakage from upstream
# changes. Upgrade deliberately and re-run the pytest + Playwright suites
# (`python -m pytest tests/ -q && python test_webapp.py`) after each bump.
#
# Reasoning for the floors:
#   - PyMuPDF 1.24: get_text("dict") line-bbox shape we rely on for qboxes
#   - Flask 3.0:   modern routing + Werkzeug 3 secure_filename
#   - anthropic SDK 0.40: max_retries kwarg on Anthropic() constructor
#   - markdownify 0.11 / beautifulsoup4 4.12: wiki-page scrape helpers
#   - requests 2.31: TLS defaults that match scioly.org cert chain
#   - gunicorn 21.2: production WSGI server for hosted deployments (Linux
#     only — local Windows dev keeps using `python review_app.py`)
#   - pytest 7.4: tests/ collection
#
# playwright is dev-only (cookie refresh for download_event.py --reauth) —
# see requirements-dev.txt. It's never imported by review_app.py at runtime,
# so the production server doesn't need it installed.
#
# doc_convert.py (.docx/.doc -> PDF normalization) shells out to LibreOffice
# headless rather than using a pip package — install it as a SYSTEM package,
# not via pip:
#   RHEL/Fedora:   sudo dnf install libreoffice-headless
#   Debian/Ubuntu: sudo apt install libreoffice
# Skippable if this event's tests are all PDF already; .docx/.doc test/key
# uploads and scrapes will fail loudly (not silently) with install
# instructions if `soffice` isn't on PATH when one is encountered.
#
# No upper bounds — that's the user's choice.
